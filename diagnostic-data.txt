Broken / At Risk
1) Third-party stream domains remain a runtime availability risk.
- Affected domains include `vidsrc-embed.ru`, `multiembed.mov`, `godriveplayer.com`, `embedmaster.link`, and `vidnest.fun`.
- Impact: button clicks can still fail when upstream providers are down or blocked regionally, even when local routing is correct.
- Current mitigation: deterministic server fallback exists, but if all providers fail, playback fails.

2) One-time migration intentionally drops malformed legacy media items with no resolvable provider/type/id key.
- Source: canonical dedupe only keeps entries with a valid canonical key.
- Impact: corrupted historical entries may disappear after migration (favors correctness over keeping broken records).

3) Full browser automation coverage is not present in this repo for desktop/mobile click-paths.
- Impact: viewport/click regressions are still possible between manual checks.
- Current status: static smoke checks and syntax validation were completed, but no Playwright/Cypress suite exists in-project.

Potential Future Issues

- TMDB fallback calls use a public key in frontend code paths.
- Risk: key rotation/quota abuse can degrade fallback behavior.

2) Dual-cloud persistence model complexity (`data-api` + Firestore mirror).
- Current behavior is stable, but long-term schema drift between operation sync and snapshot mirror paths is a maintenance risk.

3) Cross-domain dependency surface is broad.
- Detected external hosts: `storage-api.watchbilm.org`, `data-api.watchbilm.org`, `api.themoviedb.org`, `graphql.anilist.co`, `image.tmdb.org`, `youtube.com`, and multiple streaming domains.
- Risk: DNS/CORS/outage changes outside app control can cause intermittent failures.
